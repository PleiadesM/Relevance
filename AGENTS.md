# AGENTS.md

Working rules for coding agents (Claude Code / Codex / others) in this repo.

1. **Read first**: `CLAUDE.md`, then `skills/newsdash/SKILL.md` (the Page Skill —
   it defines the interview, source-intake, and secrets-guidance workflows),
   then `docs/DATA_CONTRACT.md` before touching pipeline or frontend.
2. **Safety rules (hard block)** — the full list lives in
   `skills/newsdash/SKILL.md`; the non-negotiables:
   - no secrets, tokens, capability URLs, or decrypted private data in commits, logs,
     PR bodies, or issue comments (public repos have public logs);
   - no weakening of crypto params; no plaintext fallback for private sections;
   - keep the setup-from-issue owner guard and the private-source URL ban in
     the schema;
   - the zero-secret build must always pass.
3. **Validate before claiming done**:
   `python -m pytest -q` · `python scripts/validate_config.py` ·
   `python scripts/build.py --output-dir /tmp/nd --smoke` ·
   `node tests/test_crypto_webcrypto.mjs`.
4. **Docs follow code**: if you change behavior, update
   `docs/CONFIG_REFERENCE.md` / `docs/DATA_CONTRACT.md` and their `.zh.md`
   twins in the same change.
5. `data/` is generated by the update workflow — never hand-edit or "fix" it.
